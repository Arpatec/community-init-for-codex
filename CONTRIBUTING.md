# Contribution Guide

Merci de vouloir faire evoluer ce repo.

L'objectif du projet est de rester simple, pratique et directement reutilisable dans de vrais repositories.

## Ce qui est recherche

- des ameliorations concretes de lisibilite
- des instructions plus robustes pour les agents
- des exemples d'usage verifies
- des ajustements qui reduisent la perte de contexte entre sessions
- des contributions qui restent compatibles avec des projets existants

## Ce qu'il faut eviter

- sur-architecturer le systeme
- multiplier les couches sans gain clair
- imposer des conventions qui ecrasent celles du projet cible
- transformer la memoire projet en documentation verbeuse

## Bonnes pratiques

- documenter les faits verifies, pas les suppositions
- garder `AGENTS.md` stable et court
- garder `.codex/project-memory/` concise et actionnable
- preferer des changements petits mais utiles
- conserver l'attribution au projet d'origine dans les redistributions

## Quand ouvrir une issue ou une PR

Ouvrir une issue si vous voulez :

- proposer un nouveau workflow
- signaler une ambiguite dans une skill
- montrer un cas reel ou la memoire devient obsolete trop vite

Ouvrir une PR si vous apportez :

- une correction precise
- une amelioration de documentation
- un meilleur prompt ou une meilleure instruction
- un exemple d'integration utile a la communaute

## Style du projet

- rester pragmatique
- rester lisible
- rester compatible avec des repos tres differents
- viser un systeme facile a reprendre par un autre agent plus tard
